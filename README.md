## вңҰ What is CampusMind?

CampusMind is an **AI-powered campus assistant** built for college students. It remembers your name, your branch, your interests, your clubs вҖ” and gets smarter with every conversation. Think of it as a brilliant senior student who's always available, never forgets anything you've told them, and genuinely wants to help.

```
  Student: "I'm in 3rd year CSE, interested in ML"
       вҶ“
  CampusMind remembers this forever
       вҶ“
  Next chat: "Hey! Since you're into ML, there's a Kaggle
              competition this Friday вҖ” want tips to prep?"
```

## вңҰ Features

<table>
<tr>
<td width="50%">

### рҹ§  Persistent Memory Bank
Every conversation is saved and displayed in a real-time **Memory sidebar**. CampusMind pulls context from your past chats to personalise every response вҖ” your branch, interests, clubs, and goals are always remembered.

</td>
<td width="50%">

### рҹ’¬ Full Chat History
Every conversation is persisted to **Supabase** with session tracking. Switch between past chats from the sidebar вҖ” click any session to instantly reload the full conversation exactly as it was.

</td>
</tr>
<tr>
<td width="50%">

### вҡЎ Real-time Streaming
Powered by **Groq's LLaMA 3.3 70B** вҖ” one of the fastest LLMs available. Responses stream token-by-token in real time with animated typing indicators so you always know the AI is thinking.

</td>
<td width="50%">

### рҹ”җ Secure Authentication
Email-based login via **NextAuth v4** with JWT sessions. New accounts are created automatically on first login вҖ” no sign-up form needed. All secrets secured server-side, never exposed to the browser.

</td>
</tr>
<tr>
<td width="50%">

### рҹҺ“ 4-Step Onboarding
A beautiful multi-step profile setup collects your **name, year, branch, interests, and clubs** (16 clubs available). This data seeds your memory bank so CampusMind knows you from message one.

</td>
<td width="50%">

### рҹ—„пёҸ Supabase Database
Full relational schema with **students, chat_sessions, chat_messages,** and **campus_events** tables. Row Level Security enabled. Service role key used server-side only вҖ” anon key safe for the browser.

</td>
</tr>
</table>

## вңҰ Getting Started

### Prerequisites
- Node.js 18+
- A [Supabase](https://supabase.com) project
- A [Groq](https://console.groq.com) API key

### 1 вҖ” Clone & Install

```bash
git clone https://github.com/RoNnY125-coder/campus-mind.git
cd campus-mind
npm install
```

### 2 вҖ” Set up environment variables

```bash
cp .env.local.example .env.local
```

Open `.env.local` and fill in your values:

```env
# Groq вҖ” https://console.groq.com
GROQ_API_KEY=your_groq_api_key

# Supabase вҖ” https://supabase.com/dashboard вҶ’ Settings вҶ’ API
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key   

# NextAuth
NEXTAUTH_SECRET=your_random_secret                # openssl rand -base64 32
NEXTAUTH_URL=http://localhost:3000
```

### 3 вҖ” Set up Supabase schema

Run this SQL in your **Supabase SQL Editor**:

```sql
create extension if not exists "uuid-ossp";

create table public.students (
  id            uuid primary key default uuid_generate_v4(),
  email         text unique not null,
  name          text,
  year          text,
  branch        text,
  interests     text[] default '{}',
  clubs         text[] default '{}',
  has_onboarded boolean default false,
  created_at    timestamptz default now(),
  updated_at    timestamptz default now()
);

create table public.chat_sessions (
  id         uuid primary key default uuid_generate_v4(),
  student_id uuid references public.students(id) on delete cascade,
  title      text,
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

create table public.chat_messages (
  id         uuid primary key default uuid_generate_v4(),
  session_id uuid references public.chat_sessions(id) on delete cascade,
  student_id uuid references public.students(id) on delete cascade,
  role       text check (role in ('user','assistant')) not null,
  content    text not null,
  created_at timestamptz default now()
);

alter table public.students      enable row level security;
alter table public.chat_sessions enable row level security;
alter table public.chat_messages enable row level security;

create policy "Service role all" on public.students      for all using (true) with check (true);
create policy "Service role all" on public.chat_sessions for all using (true) with check (true);
create policy "Service role all" on public.chat_messages for all using (true) with check (true);
```

### 4 вҖ” Run locally

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) вҖ” enter any email to create your account.

## вңҰ How the Memory System Works

```
User sends message
       в”Ӯ
       в–ј
  /api/chat receives it
       в”Ӯ
       в”ңв”Җв”Җв–ә Groq LLaMA 3.3 generates response (streaming)
       в”Ӯ
       в”ңв”Җв”Җв–ә Response saved to chat_messages (Supabase)
       в”Ӯ
       в””в”Җв”Җв–ә Memory sidebar polls /api/memory every 8s
                   в”Ӯ
                   в””в”Җв”Җв–ә Reads latest assistant messages
                         from chat_messages table
                         вҶ’ displays as memory cards
                         вҶ’ new cards flash green on arrival
```
